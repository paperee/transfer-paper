## 指令awk入门

> Date: 2026.06.22 | Author: PAPEREE

大部分人听说过`awk`之难，令人望而却步。

实则不然，它的基本语法极其简洁而规律：

```bash
awk '条件 {动作}' 文件名
```

其中，**条件**是可选的，跳过则不对选中行进行任何条件判断。

**动作**也是可选的，跳过则打印当前整行。

以下是三大条件运算符，我们将在后面使用它们：

- **逻辑运算符**：与`&&`、或`||`、非`!`。
- **关系运算符**：`==`、`!=`、`>`、`<`、`>=`、`<=`。
- **正则匹配运算符**：包含`~`、不包含`!~`。

`awk`在按行读取文本时，会为该行文本切片，并存入内置变量中：

- `$0`：代表**整行**文本。
- `$1、$2、$3...`：代表第`1、2、3...`列。默认以**空格**或**制表符**来分隔。
- `NF` (_Number of Fields_)：当前行的**总列数**。可用`$NF`代表**最后列**。
- `NR` (_Number of Records_)：当前处理的**行号**，从1开始计数。

### 1. 例题与基本语法实践

假设有一个名为`博客相关.txt`的文件，你可以通过如下操作写入：

```bash
echo '类型       最新    历史
运行天数    6      57
总访问量    201    1145
总评论数    625    625
总文章数    62     62' > 博客相关.txt
```

假如，我们想**跳过表头**，然后只查看**类型**列（**第1列**）和**历史**列（**最后列**）的数据。

很显然，我们需要一个**条件**：`NR > 1`，表示行号大于1，即**跳过表头**；

接下来需要一个**动作**：`print $1, $NF`，表示打印**第1列**和**最后列**。

#### (1) 排列组合

```bash
awk 'NR > 1 {print $1, $NF}' 博客相关.txt
```

各司其职，顺利输出：

```bash
运行天数 57
总访问量 1145
总评论数 625
总文章数 62
```

> `awk`指令应该使用**单引号**进行强引用，而不是**双引号**。
> 防止以`$`开头的**内置变量**在传递给`awk`前被`shell`当作**环境变量**解析。

#### (2) 更进一步

如果介意两列之间只有单个空格，还可以在**动作**中插入`\t`制表符：

```bash
awk 'NR > 1 {print $1"\t"$NF}' 博客相关.txt
```

输出如下：

```bash
运行天数        57
总访问量        1145
总评论数        625
总文章数        62
```

> 在`awk`的语法中，不加双引号的字母组合会被视为**内置变量**。
> 因此，此处的`\t`制表符需要使用双引号包裹，代表**字符串常量**。

了解以上的基本语法后，我们就可以读懂以下几个案例。

### 2. 带上行号与自定文本

```bash
awk '{print "第"NR"行", $1}' 博客相关.txt
```

注意，这里被**双引号**包裹的是`第`字和`行`字，输出：

```bash
第1行 类型
第2行 运行天数
第3行 总访问量
第4行 总评论数
第5行 总文章数
```

### 3. 进行简单的数学计算

我们希望将**运行天数**和**总访问量**的，**最新**列与**历史**列相加。

（_由上一个案例_）可知它们的行数分别是2和3，于是：

```bash
awk 'NR == 2 || NR == 3 {print $1, $2 + $3}' 博客相关.txt
```

注意理解此处**或**`||`的写法。

```bash
运行天数 63
总访问量 1346
```

### 4. 自定分隔符与模糊匹配

我们可以使用`-F`参数来指定分隔符，例如：

```bash
awk -F '总' '{print $NF}' 博客相关.txt
```

表示见到`总`字才进行切片，然后使用`$NF`取**最后**的片段。

```bash
类型       最新    历史
运行天数    6      57
访问量    201    1145
评论数    625    625
文章数    62     62
```

### 5. 选修之BEGIN与END

`awk`还有两个非常强大的**特殊条件**：

- `BEGIN`：在读取文件之前执行，_常用于画表格头_。
- `END`：在读取完文件所有内容之后执行，_常用于算总和、跑平均数_。

例如、让我们做一件无意义的事：

将**历史**列的所有数字**累加**，做奇怪的运算。


### 6. 图灵完备的编程语言

`awk`是图灵完备的！_这听起来是不是很令人惊讶？_

它支持：

- **控制流**：`if/else`条件分支、`for`和`while`循环；
- **状态存储**：使用关联数组/字典，变量默认初始值为0或空字符串；

## 指令grep入门

> Date: 2026.06.25 | Author: PAPEREE

`grep`比`awk`好学得多，它的基本语法直截了当，与Linux大多数指令类似：

```bash
grep [选项] '关键词' 文件名
```

不难得知，`grep`的强大之处在它的**参数**（选项）中。

以下是**最常用**的四个参数：

- `-i`：_大写小写，我全都要_（**忽略大小写**）
- `-v`：_反向选择，排除自己_（**排除**）
- `-n`：_带上行号，精准定位_（**行号**）
- `-c`：_不关心内容，只打印行数_（**计数**）

背下来没意思，熟能生巧，用起来才算本事。

为了方便运行后面的测试指令，我们创建并进入测试文件夹：

```bash
mkdir test && cd test
```

接着，让我们创造一批文件夹，在每个文件夹中放一些有字的文件！

```bash
for i in {0..9}; do mkdir "$i"; for i2 in {0..19}; do echo -e "\n$i2"{a..z} > "$i/$i2"; done; done
```

这里使用到了`for`**嵌套循环**和`{}`**花括号展开**，但它们不是本节课的重点。

没学过也不要感到害怕（）让我们忽略它的实现细节（

下面是**最基础**的无参数`grep`指令：

```bash
grep '0' 0/0
```

**注意**：这里搜索的是`0/0`这个**文件**中包含`0`的**行**。输出：

```bash
0a
0b
0c ……
```

（_仅截取部分内容，使用省略号，下同_）

接下来，我们会通过一些更有趣的案例来加深对`grep`强大功能的认识。

### 1. 多文件与递归搜索

- `-r`：_深入目录，递归搜索_（**遍历子目录**）
- `-l`：_打印文件名，不关心内容_（**文件名**）

刚才的`for`循环创建了`0~9`的文件夹，又在每个文件夹中创建了`0~19`的文件。

但是，我们如果想考究这个操作是否符合预期，就应该**检查**：

- _文件是否放在正确的目录_。
- _文件数量是否正确_（10x20=200）。

诚然，我们可以使用`ls -R`来递归罗列所有子文件夹中的文件。

也可以使用`grep`，搭配`-r`参数**遍历子目录**、`-l`参数**打印文件名**，如：

```bash
grep -rl '' * > 测试.txt && cat 测试.txt
```

此处的`''`代表**空关键词**，即*匹配所有内容*；`*`是**通配符**，代表*当前任意文件/文件夹*。

`> 测试.txt && cat 测试.txt`只是将输出写入文件并打印，方便下文沿用。

输出：

```bash
0/3
0/0
0/12 ……
```

Perfect！接下来，我们应该对它打印的目录做数量上的统计。

还记得吗？上文提到`grep`的四个常用**参数**，其中`-c`表示**计数**。

```bash
grep -c '' 测试.txt
```

终端输出`200`，即共有`10x20=200`个文件！符合我们的预期。

### 2. 不伤及无辜的参数

继续来看`测试.txt`文件。

#### (1) `-x`：精准匹配整行

默认情况下，`grep`会匹配行内的任意子串。

例如，当我们只想查找整行刚好为`1/1`的目录时：

```bash
grep '1/1' 测试.txt
```

输出将包含许多干扰项：

```bash
1/12
1/11
1/10 ……
```

加上`-x`参数后，`grep`要求整行内容必须与模式完全一致：

```bash
grep -x '1/1' 测试.txt
```

终端输出`1/1`，不错！

#### (2) `-w`：精准匹配整个单词

如果我们只想过滤出名称恰好为`2`的文件：

```bash
grep '2' 测试.txt
```

直接使用默认匹配，输出将鱼龙混杂：

```bash
0/12
0/2
2/3 ……
```

不仅出现了`12`等不严格包含数字`2`的**文件**，还出现了`2/`这个**目录**。

既然如此，我们需要`-w`参数，来**限定**独立的数字`2`。

在Linux的定义中，斜杠被视为**分隔符**，`2/`目录还会出现在结果中。

因此，还需要通过**管道符**叠加`-v`参数，来**排除**路径中带有`2/`的行。

```bash
grep -w '2' 测试.txt | grep -v '2/'
```

最终输出：

```bash
0/2
1/2
3/2 ……
```

### 3. 使用正则表达式

书接上节，不难发现，`2/2`这个目录也被排除在外。

有没有更完美的优雅解法？当然有，那就是**正则表达式**！

> 正则表达式是一套强大的**文本匹配规则**。
> 如果你是第一次接触，请看：https://regexlearn.com/

回到我们的问题，将正则的**锚定符**塞进去试试：

```bash
grep '/2$' 测试.txt
```

> 这里的`$`是正则中的特殊符号，表示**行尾**。
> 而`/2$`，就代表只匹配以`/2`结尾的**行**。

精准输出：

```bash
0/2
1/2
2/2 ……
```

> 然而，普通的`grep`只支持BRE基础正则。
> 该模式存在许多不得不加**反斜杠**的**反人类写法**。

<f0>~~以上不作展开，请在实践中自行领会，或者查阅资料。~~</f0>

#### (1) `-E`：开启ERE拓展正则

让我们来看一个稍微复杂的正则组合：

```bash
grep -E '^[5-7]/1+$' 测试.txt
```

这个表达式的意思是：

- `^`：匹配**行首**。
- `[5-7]`：开头必须是`5~7`中的数字。
- `/1+`：斜杠后跟着**至少1个**数字1。
- `$`：匹配**行尾**。

输出结果：

```bash
5/11
5/1
6/11 ……
```

无论走到哪里，**正则表达式**永远是**文本搜索**的终极大招。

#### (2) `-P`：开启PCRE高级正则

> Perl语法拓展了`\d`、`\w`、`\s`等**字符集**以及更多**现代化特性**。
> 如果发现**合规正则**无法在`-E`下正常运作，可以尝试`-P`参数。

### 2. 拓展：命令行参数标准

对于jmr指令中`-F:`和`-k1`这样**紧挨**的写法，这里放置*选读内容*。

Linux参数通常分为**短参数**（单横杠）和**长参数**（双横杠）

根据它们是否需要接收值，可以分为两种类型：

#### (1) 布尔型：不需要具体值

这类参数就像一个**开关**，只要携带就会立马切换`true`状态。

如果有多个**布尔型**参数，它们可以很轻松地挤在一起写。

例如前面用到的`grep -rc`，`-rc`其实是`-r`和`-c`的连用。

#### (2) 入参型：必须带尾巴

这类参数就像一个**函数**，需要给它传递参数值才能吐出东西。

- **短参数**：可以**不加空格**连着写，例如`awk -F:`。
- **长参数**：必须用**等号**连接，例如`awk --field-separator=:`。

#### (3) 布尔型与入参型混用

在这个情况下，**入参型**必须唯一，且连同它的尾巴一起放在**最后一位**。

否则**入参型**后面携带的参数都会被误认为是它的尾巴。
